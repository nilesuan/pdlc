# 02-plan-exit.md — Phase 02 (Plan) exit checklist

**Authoritative source:** [`../../../handbook/02-plan.md`](../../../handbook/02-plan.md).

This checklist is run at the end of `/plan`.

## Done-when

- [ ] **Strategy kernel written** (Rumelt): diagnosis, guiding policy, coherent actions.
- [ ] **MVP scope explicit** — list of in-scope items AND a NOT-in-scope list of comparable specificity.
- [ ] **OKRs drafted** — 1–3 Objectives; 3–5 outcome-based Key Results per Objective. KRs are measurable, time-bound, and not output proxies (no "ship feature X" KRs).
- [ ] **Now / Next / Later roadmap** drafted (no firm dates beyond "Now"). Quarterly granularity max for "Next"; "Later" is themes.
- [ ] **Backlog initialized** with RICE / WSJF or equivalent prioritization signal on top items.
- [ ] **Sprint cadence chosen** (2-week default). First sprint goal stated.
- [ ] **Estimation approach documented** (story points, t-shirt, #NoEstimates) — and *why* this fits the team.
- [ ] **Risks register** with mitigation owner per risk.
- [ ] **Stakeholders identified** with engagement model (informed / consulted / accountable).
- [ ] **Definition of Done** for the team's work agreed.

## Auto-rejection

| Trigger | Severity |
|---|---|
| OKRs with output-proxy KRs ("ship feature X") | Major |
| > 3 Objectives or > 5 KRs per Objective | Major (focus dilution) |
| Roadmap has firm dates beyond the current quarter | Major |
| MVP scope without an explicit NOT-in-scope list | Major |
| Backlog with no prioritization signal | Major |
| Risks register with no owners | Minor |
| "Velocity" used as a target rather than a forecasting input | Minor |

## What good looks like

- The OKRs would let an outsider tell whether you succeeded next quarter from public metrics, without asking the team.
- The MVP NOT-list is roughly the same length as the in-scope list and has explicit deferral reasons ("will revisit when X learnings come back").
- The strategy kernel passes the "could you say the opposite?" test — if the diagnosis or guiding policy is so generic that nobody could disagree, it's not strategy.
- The roadmap distinguishes commitments (Now) from intent (Next) from speculation (Later) — and they look different in the doc.
